Flask + Celery + Redis + Supabase

Esta aplicación en Flask pone en práctica conceptos de gestión de grandes volúmenes de datos mediante una arquitectura asíncrona y optimizada.

Redis actúa como cola de procesamiento y sistema de caché, agrupando temporalmente las solicitudes antes de volcarlas en la base de datos PostgreSQL (gestionada con Supabase).

Celery gestiona las operaciones de lectura y escritura mediante workers, optimizando inserciones y consultas SQL de forma asíncrona y escalable.

Cada minuto, los trabajadores transfieren los datos de Redis en lotes de 5 registros a PostgreSQL siguiendo una estructura FIFO y actualizan en caché los rankings de Top Users y Top URLs.

Los usuarios pueden ver las URLs que han añadido; cada nuevo registro se almacena temporalmente hasta que un worker lo inserta definitivamente en la base de datos.
